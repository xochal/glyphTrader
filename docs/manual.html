1. Introduction

What is glyphTrader

glyphTrader is a standalone, containerized, automated stock trading system built on the TFT V4/V7 strategy — a 100% technical analysis approach with zero AI or machine learning. It connects exclusively to Tradier for both trade execution and market data, runs entirely inside Docker containers on a single server, and provides a modern web interface for monitoring, configuration, and manual trade management.

The system evaluates 64 stocks across 20 sectors daily, scoring each with a composite technical indicator (the V4 Score), filtering candidates through a rigorous 7-step cascade, and placing bracket orders automatically. Positions are managed with ATR-based stops and targets, stepped stop ratchets, pyramiding rules, and time-based exits.

For traders who also hold positions outside the automated strategy, glyphTrader offers full manual trade management: discover, adopt, and manage orphan positions with configurable stop/target modes, ratcheting stops, and hold mode.

Key Features

  • Fully Automated Trading — Daily scoring, filtering, entry, and position management with no manual intervention required
  • Manual Trade Management — Adopt broker-side positions, set custom stops/targets, hold mode, release or close at will
  • 6-Layer Data Freshness Guards — Never trades on stale data
  • Bracket Orders (OTOCO) — Every entry includes stop loss and OCO profit targets
  • ATR-Based Risk Management — Volatility-adaptive stops, targets, and position sizing
  • Market Regime Awareness — VIX and benchmark SMA filters prevent trading in hostile markets
  • One-Command Deployment — Docker Compose with automatic HTTPS via Caddy
  • Encrypted Credentials — Fernet encryption with Argon2id key derivation
  • Observe-Only Mode — Monitor without placing trades
  • Kill Switch + Flatten All — Emergency controls always accessible from the dashboard
  • Paper Trading by Default — Sandbox mode enforced unless a version-locked production license key is activated

2. Installation

Linux Quick Start

On a fresh Linux server (Ubuntu 20.04+ or similar), the entire installation takes three commands: 

git clone https://github.com/xochal/glyphTrader.git
cd glyphTrader
chmod +x setup.sh && ./setup.sh

The setup script handles everything: Docker installation, configuration, container builds, and first-run initialization. You will be prompted to set an admin password during setup.

What setup.sh Does

  1. Docker Check/Install — Detects if Docker and Docker Compose are installed; installs them if missing
  2. Admin Password Prompt — Asks you to create a secure admin password (used for all logins)
  3. Port Detection — Checks for port conflicts on 80/443 and configures Caddy accordingly
  4. Container Build — Builds and starts all three containers (Caddy, backend, frontend)
  5. Recovery Key Display — Generates and displays a one-time recovery key — save this immediately
Critical

Save your recovery key immediately after setup. It is the only way to regain access if you forget your admin password. Store it in a password manager or secure offline location.

macOS / Windows Manual Setup

For macOS or Windows (with WSL2), install Docker Desktop first, then:

git clone https://github.com/xochal/glyphTrader.git
cd glyphTrader
cp .env.example .env
# Edit .env to set your desired ports if needed
docker compose build
docker compose up -d

On first startup, you will need to access the web interface and complete initial password setup.

Accessing the Web Interface

Once the containers are running, open your browser and navigate to: 

https://localhost

Caddy generates a self-signed TLS certificate automatically. Your browser will display a certificate warning — this is expected and safe on your own server. Click through to proceed.

Note

If you configured a custom domain, Caddy will automatically obtain a valid Let's Encrypt certificate and the browser warning will not appear.

Saving Your Recovery Key

The recovery key is displayed once during setup.sh execution. To use it later, send a POST request to the recovery endpoint: 

POST /api/auth/recover
Content-Type: application/json
{"recovery_key": "your-recovery-key-here", "new_password": "your-new-password"}

This resets the admin password and re-encrypts all stored credentials with the new password's derived key.

3. First Login

Figure 3.1 — The glyphTrader login page

glyphTrader uses a single admin user model. There is one account, one password, and one active session at a time. Enter your admin password on the login page to access the system.

Session Behavior

  • Session Timeout: 10 minutes of inactivity triggers automatic logout
  • JWT Authentication: Sessions use JWT tokens with refresh token rotation for security
  • Revoke All Sessions: Available in the Settings page to force logout from all devices

System Lock vs. Unlock

glyphTrader encrypts all sensitive data (Tradier API tokens, account numbers) using Fernet encryption with keys derived from your admin password via Argon2id. This means:

  • Locked state: After a container restart, the system is locked. It can display the dashboard and historical data but cannot place trades or communicate with Tradier.
  • Unlocked state: Logging in derives the encryption key and unlocks the system. Trading becomes possible, and the scheduler begins its normal cycle.
Important

After a server reboot or container restart, you must log in at least once to unlock the encryption system. New trades will be blocked until the admin logs in.

4. Dashboard

Figure 4.1 — The Dashboard showing regime state, portfolio stats, and equity curve

The Dashboard is your primary monitoring view. It displays the current market regime, portfolio performance, and provides critical safety controls.

Header

The top navigation bar shows the application name, current page, and version number. When a newer version is available on the remote repository, an Update Available chip appears next to the version.

Safety Controls

Observe-Only Mode

The blue Observe Only button toggles observe-only mode. When enabled, the system continues its full analysis cycle (data fetch, enrichment, scoring, plan generation) but does not place any orders. This is useful for monitoring the strategy's behavior before committing real capital. A password is required to toggle this mode in either direction.

Enable / Disable Trading (Kill Switch)

The Enable Trading / Disable Trading button is the system's kill switch. When disabled, no new automated trades are placed and existing position management (stops, targets) continues. Enabling trading requires password confirmation to prevent accidental activation.

Flatten All

The Flatten All button is an emergency control that:

  1. Cancels all open orders on Tradier
  2. Market-sells all held positions immediately
  3. Marks all tracked positions as CLOSED
Warning

Flatten All is irreversible. It requires trading to be disabled (kill switch activated) and password confirmation. Use only in genuine emergencies.

Regime State

The Regime panel displays the current market environment:

  • VIX Level: Current CBOE Volatility Index value
  • SPY above SMA(100): Whether the S&P 500 ETF is trading above its 100-day simple moving average
  • QQQ above SMA(100): Whether the Nasdaq-100 ETF is trading above its 100-day simple moving average
  • TRADING ALLOWED / BLOCKED badge: Green badge when the regime filter passes, red when it blocks entries

The regime filter is checked per stock against its assigned benchmark (SPY or QQQ). Entry is allowed if the stock’s benchmark is above its 100-day SMA, or VIX is below 20. If VIX is at or above 32, all new entries are blocked regardless of benchmarks.

5. Auto Trades

Figure 5.1 — Auto Trades page (no positions currently open)

How the Automated Strategy Works

Each trading day, glyphTrader executes a fully automated cycle:

  1. 12:30 PM ET — Fetch daily OHLCV bars from Tradier and VIX data from yfinance, then enrich with all technical indicators and compute V4 scores
  2. 12:45 PM ET — Apply the 7-filter cascade and generate entry plans for qualifying stocks
  3. 1:00 PM ET — Execute trades by placing bracket orders (OTOCO) on Tradier
  4. Every 2 minutes — Monitor fills, manage stops, ratchet trailing stops

6. Manual Trades

Figure 6.1 — Manual Trades showing discovered orphan positions

The Manual Trades page is designed for managing positions that exist on your Tradier account but were not initiated by glyphTrader's automated strategy. This includes positions you opened manually through Tradier's website or another platform.

Discovered Orphans

Every 5 minutes during market hours, glyphTrader's reconciliation engine compares positions on your Tradier account against its internal database. Any positions not tracked by glyphTrader appear as orphans in the Discovered Orphans section.

For each orphan, you have two options:

  • Adopt — Brings the position into glyphTrader's management system. You will be prompted to configure stop/target parameters. Once adopted, the system places protective orders and monitors the position just like an automated trade.
  • Dismiss — Hides the orphan from the list. Dismissed orphans are remembered and will not reappear unless you clear them. They auto-clear when the position is sold on Tradier.

Trade Config Modal

When you click Adopt on an orphan or Edit on an existing manual position, the Trade Config Modal appears. This dialog lets you configure how the position is managed:

Stop Mode

  • ATR-based: Stop is calculated as a multiple of ATR (Average True Range) below the entry price. The default multiplier is 3.3x ATR.
  • Fixed Dollar: Stop is set at an exact dollar price you specify.

Stop Value

For ATR mode, this is the multiplier (e.g., 3.3). For Fixed mode, this is the dollar price (e.g., 145.50).

Ratchet (Trailing Stop)

When enabled, the stop price ratchets upward as the stock price rises. The ratchet only moves up, never down, protecting accumulated gains.

  • Ratchet Mode — ATR: The ratchet maintains a fixed ATR distance below the current high
  • Ratchet Mode — Percentage: The ratchet maintains a fixed percentage below the current high

Target Mode

  • ATR-based: Targets calculated as ATR multiples above entry (T1: 0.7x, T2: 1.5x, T3: 3.0x by default)
  • Fixed Dollar: You specify exact dollar prices for each target

Exit Percentages

How much of the position to sell at each target level. The default split is 70% at T1, 20% at T2, and 10% at T3 (must total 100%).

Open Manual Positions

Once adopted, manual positions appear in the Open Manual Positions table with the same columns as auto trades plus additional action buttons:

ActionDescription
EditReopen the Trade Config Modal to change stop/target settings
Hold / TargetsToggle between Hold mode (stop-only, no targets) and Targets mode (normal T1/T2/T3 exits)
CloseCancel all orders and market-sell the entire position immediately. Irreversible.
ReleaseCancel glyphTrader's orders but keep the shares on Tradier. The position becomes untracked.

Mode Badges

Manual positions display mode badges indicating their current configuration:

  • Hold — Stop-only mode, targets disabled
  • Ratchet — Trailing stop enabled
  • Fixed — Using fixed-dollar stops/targets (not ATR-based)

Production License

glyphTrader ships in paper trading mode by default. The Production environment option in the credentials dropdown is disabled until a valid license key is activated. This is a safety measure to ensure that anyone running the system has explicitly acknowledged the risks of live trading.

How Licensing Works

  • License keys are Ed25519 digital signatures tied to a specific software version
  • Each key has the format GT-<base64-encoded-signature> (approximately 90 characters)
  • Keys are version-locked — when glyphTrader is updated to a new version, the existing key is automatically invalidated and the system reverts to sandbox mode
  • To obtain a new key after an update, contact the project maintainer, re-read the WAIVER.md, and confirm your acceptance of risk

Activating a License Key

  1. Navigate to Settings
  2. Find the Production License card (between Tradier Credentials and Change Password)
  3. Paste your license key (starting with GT-) into the input field
  4. Click Activate License
  5. If valid, the status changes to LICENSED and the Production environment option becomes available in the credentials dropdown

Removing a License

Click Remove License on the Production License card. This clears the key and immediately forces the environment back to Sandbox. A confirmation dialog prevents accidental removal.

Version Invalidation

When glyphTrader starts up after an update, it compares the stored license version against the current software version. If they differ:

  1. The stored license key is cleared
  2. The environment is forced to Sandbox
  3. A warning is logged to the backend console

This is intentional — it ensures you re-review the liability waiver and obtain a fresh key for each version, since code changes may alter trading behavior.

Recommendation

Paper trade for 30 to 90 days before requesting a production license key. This gives you time to observe the system’s behavior, verify data freshness, and build confidence in its operation.

Paper Trading Banner

When the system is running without a valid production license (or with the environment set to Sandbox), an amber PAPER TRADING MODE banner appears at the top of every page. This banner is hidden only when a valid license is active and the environment is set to Production.

Change Password

Change the admin password by entering the current password, the new password, and confirming it. Changing the password re-derives the Fernet encryption key and re-encrypts all stored credentials.

Security

The Revoke All Sessions button invalidates all active JWT tokens, forcing a logout on every device. Use this if you suspect unauthorized access.

10. How the System Works

The TFT V4/V7 Strategy

glyphTrader implements the TFT V4/V7 strategy, a 100% technical analysis approach with zero AI or machine learning. The strategy was developed and backtested over 6+ years before being ported to this automated system.

V4 Composite Score

The core of the strategy is the V4 Composite Score — a weighted blend of six technical indicators that produces a single score (0–100) measuring the quality of a stock's current technical setup.

Primary Indicators (Weighted)

IndicatorWeightWhat It Measures
Linear Regression Slope40.9Trend direction and strength over the regression window
ADX (Average Directional Index)38.9Trend strength (directional movement)
Stochastic K25.8Momentum oscillator showing current price relative to range
RSI (Relative Strength Index)22.2Momentum measuring speed and magnitude of price changes

Modifiers

  • Bollinger Band Position (BB_PCT): Measures where price sits within the Bollinger Bands, modifying the composite score
  • Volume Surge (VOL_SURGE): Detects unusual volume relative to the moving average, acting as a confirmation modifier

Overbought Penalties

  • Stochastic > 92: Applies a penalty to prevent entries into overbought momentum
  • RSI > 80: Applies a penalty to prevent entries into overbought conditions
  • Extended Penalty: A penalty of -0.953 points applied when BB%B exceeds 93.6% (price far above upper Bollinger Band)

Sector-Specific Thresholds

Different sectors require different V4 scores to trigger an entry. Higher thresholds are applied to more volatile or crowded sectors:

ThresholdSectors
68Semiconductors, Utilities
75Nuclear (SMR), Healthcare, Biotech, Technology, Cloud/SaaS, Clean Energy, Industrials, Insurance, Materials, Energy, Consumer, Consumer Staples, Mega Tech (NFLX)
78Solar
80Nuclear (BWXT, CCJ, DNN, NLR, UEC, UUUU)
82Mega Tech (AAPL, AMZN, GOOGL)

Daily Trading Cycle

The automated trading cycle runs on a fixed schedule every trading day (market holidays excluded):

12:30 PM ET
Data Fetch — Downloads daily OHLCV bars from Tradier for all 64 stocks plus SPY/QQQ benchmarks. VIX historical data is fetched from yfinance (Tradier does not provide VIX history). Data is stored in SQLite for persistence across restarts.
12:45 PM ET
Plan Generation — Runs the 7-filter cascade on all scored stocks. Qualifying signals become entry plans. Aborts if data is stale, benchmarks are missing, or VIX data is unavailable.
1:00 PM ET
Trade Execution — Executes pending entry plans as bracket orders (OTOCO) on Tradier. Processes time stops first, then places new orders with calculated position sizes.
Every 2 min
Monitor Cycle — Checks for order fills, updates stop levels (stepped stops, breakeven, trailing ratchet), manages target exits, and enforces time stops.
Every 5 min
Reconciliation — Compares Tradier positions against internal records. Detects ghost positions (tracked but not on broker), orphans (on broker but not tracked), and quantity mismatches. Cleans up stale orders and updates position states.
4:05 PM ET
End-of-Day Snapshot — Records portfolio value, benchmark prices (SPY, QQQ), and daily P&L for the equity curve and performance tracking.

The 7-Filter Cascade

Every potential entry must pass all seven filters in this exact order. If a stock fails any filter, it is rejected and the remaining filters are not evaluated:

1
V4 Score — The stock's V4 composite score must meet or exceed its sector-specific threshold (68–82 depending on sector).
2
EMA Crossover — The 5-period EMA must be above the 13-period EMA, with close_only confirmation (EMAs are calculated on closing prices).
3
Price Movement — The overnight price change must be less than 0.6x ATR. This blocks entries on stocks that gapped significantly, where the technical setup may be unreliable.
4
Slippage — The gap between the signal price (when the score was calculated) and the current market price must be within 5%. This filter blocked 43 entries over 6 years of backtesting and is often overlooked.
5
Regime — Each stock is checked against its assigned benchmark index (SPY or QQQ, per watchlist config). Entry is allowed if: (a) the stock’s benchmark is above its 100-day SMA, or (b) VIX is below 20. If VIX is at or above 32, all entries are blocked regardless of benchmarks.
6
Sector — Stocks in Tier 5 sectors are blocked from entry. Currently, no stocks in the universe are Tier 5, but this filter exists as a safety mechanism.
7
Cash — Sufficient capital must be available for the position size. The required amount is calculated based on the position sizing percentage, VIX sizing multiplier, and current account value.

Position Management

Entry: Bracket Orders (OTOCO)

Every entry is placed as an OTOCO (One-Triggers-OCO) bracket order on Tradier:

  • Primary order: Market or limit buy for the calculated share quantity
  • Stop loss: Sell stop at 3.3x ATR below entry price
  • OCO targets: Limit sells at T1, T2, and T3 prices with appropriate quantities

When the primary order fills, the protective orders activate automatically.

ATR-Based Targets

All stops and targets are calculated relative to the 14-period ATR (Average True Range) at the time of entry, using a 70/20/10 exit split:

LevelDistanceExit %Example (Entry $100, ATR $2.00)
Stop3.3x ATR below100% (remaining)$93.40
T10.7x ATR above70%$101.40
T21.5x ATR above20%$103.00
T33.0x ATR above10%$106.00

Pyramiding

If a stock continues to score well after entry, up to 2 pyramid adds (3 total tranches) are allowed, each requiring:

  • V4 score of at least 75 at time of pyramid
  • Available cash for the pyramid allocation (8% of account by default)

On each pyramid add, all stops and targets are reset to the new blended average entry price. This ensures risk management reflects the actual cost basis, not the original entry.

Stepped Stops

Each trading day, the stop-loss price is ratcheted upward by a calculated step: 

new_stop = base_stop + (active_days x 0.5 x ATR)

The stepped stop is capped at the breakeven level before T1 fills, preventing the stop from rising above the entry price prematurely. After T1 fills, the stop continues stepping upward without a cap.

Breakeven

When T1 fills (the first profit target is hit), the base stop is raised to entry price + 1.0%, locking in a small profit on the remaining shares. This breakeven stop is capped at T1 price minus 1.0% to prevent the stop from sitting too close to the fill zone. When T2 subsequently fills, the stop is pulled up further to the T1 price level, protecting the bulk of the gains.

Trailing Stops (Ratchet)

After the position reaches maximum size (all pyramid slots used), the stop becomes a trailing ratchet that only moves up, never down. This locks in progressively higher profits as the stock advances.

Time Stops

  • Hard time stop: After 60 calendar days, the position is exited regardless of P&L
  • Stagnant win exit: If T1 has been hit and the trade has been held for 20+ days with less than 5% total profit, the remaining shares are sold

Data Freshness (6 Safety Layers)

glyphTrader never trades on stale data. Six independent safety layers prevent this:

LayerCheckDescription
1 Fetch Threshold Enrichment aborts if fewer than 75% of stocks were fetched successfully, or if benchmark data (SPY/QQQ) is missing
2 DataStore Clearing Old in-memory data is wiped before each enrichment cycle, preventing stale symbol data from persisting
3 Global Freshness Plan generation checks that the DataStore was updated today before generating any entry plans
4 VIX Staleness VIX data must be no more than 4 calendar days old (covers weekends plus one holiday)
5 Per-Symbol Staleness Each stock and benchmark is individually verified for current data before use
6 NaN Guards V4_SCORE, SMA_100, and ATR_14 are explicitly checked for NaN before any calculation or trade decision

Design Constraints

  • Single server, no failover: Simple deployment, no distributed coordination
  • Single worker (--workers 1): Required because Fernet key cache is per-process
  • Integer cents: All monetary values stored as integers (e.g., $150.25 = 15025) to eliminate floating-point rounding errors
  • Whole shares only: Fractional shares are not supported because GTC OCO bracket orders require whole share quantities
  • Tradier only: Single broker for both execution and market data (no Alpaca, no other brokers)
  • SQLite only: Portable, zero-configuration database (no PostgreSQL)
  • Form-encoded API: Tradier API requires form-encoded POST bodies, not JSON

Security

  • Fernet encryption with Argon2id key derivation for stored credentials
  • JWT authentication with refresh token rotation
  • Rate limiting on authentication endpoints
  • Audit logging for all security-relevant events
  • Observe-only mode for safe monitoring
  • Kill switch with password confirmation
  • Session revocation for compromised access
  • Self-signed HTTPS by default (automatic Let’s Encrypt certificates with custom domains)
  • Production license lock with Ed25519 signature verification and version-bound keys

11. Updating

Linux (One Command)

./update.sh

The update script performs the following steps automatically:

  1. Pulls the latest code from the remote git repository
  2. Rebuilds Docker images with the new code
  3. Restarts containers with the updated images
  4. Runs health checks to verify all services started correctly

macOS / Windows

git pull
docker compose build
docker compose up -d

Data Preservation

Your data survives updates because:

  • SQLite database files are stored on a bind-mounted volume (./data:/app/dbdata) outside the container filesystem
  • Container recreation does not touch mounted volumes
  • Schema migrations are applied automatically on startup when the database version is behind
Safe

Updating never deletes your trade history, positions, settings, or credentials. The SQLite database persists across all updates and container rebuilds.

License Reset

After an update, your production license key is automatically invalidated and the environment reverts to Sandbox. You must obtain a new license key for the updated version before switching back to Production. This is a safety measure — code changes may affect trading behavior, so each version requires explicit re-acknowledgment of the risks.

13. How We Built It

glyphTrader was designed by Russell Shipe and co-developed with Claude, Anthropic's AI assistant, using Claude Code (Anthropic's CLI for Claude).

Design Philosophy

The project followed a rigorous design-first approach: the complete BUILD_PLAN.md specification (1,100+ lines) went through four rounds of design review before a single line of code was written. A separate REVIEW_FINDINGS.md documented known pitfalls and bugs to avoid during implementation.

PhaseNameDescription

2Data PipelineTradier client, yfinance VIX, technical indicators, V4 scoring, DataStore
3Signal Engine7-filter cascade in exact order, plan generation, entry logic
4Position ManagementBracket orders, target exits, stepped stops, pyramiding, time stops

6HardeningError handling, logging, scheduler, deployment scripts, security audit
7Manual TradesManual Trades page, reconciliation, orphan adoption, configurable stops/targets, ratchets, hold mode, flatten all

Key Technical Decisions

  • Tradier-only: Unified execution and market data from a single broker, eliminating multi-vendor complexity and data reconciliation issues
  • SQLite: Zero-configuration, portable database that ships with the application. WAL mode provides concurrent read access during writes.
  • Docker: One-command deployment with setup.sh, portable across any Linux server
  • Integer cents: Eliminates floating-point rounding errors that plague financial calculations. All arithmetic is exact.
  • Whole shares: Tradier's GTC OCO bracket orders require whole share quantities. No fractional share workarounds needed.
  • APScheduler inside FastAPI: No external cron jobs. The scheduler runs in the same process, ensuring coordination with the trading state machine.

Strategy Heritage

The TFT V4/V7 strategy was ported from a production trading system with over 6 years of backtested data. The V4 scoring weights, filter cascade order, and position management rules were preserved exactly as designed, with no re-optimization or curve fitting during the port.

Quality Assurance

The codebase was hardened through multiple comprehensive audits:

  • Security review: Credential encryption, JWT handling, rate limiting, audit logging, input validation
  • Code correctness review: V4 scoring fidelity, filter cascade order, position management math, money handling (integer cents), edge cases
  • Live functionality testing: Tradier sandbox integration, order flow, reconciliation, recovery from restarts
  • Data freshness audit: All 6 staleness guards verified against real-world scenarios (weekends, holidays, API failures)
